## Project title

**Kalshi 15-Minute Sell Bot — Advanced Dry-Run Simulator**

## One-line description

A large Python market-automation study that brings positions, orders, fills, fees, and live market data into one reviewable sell-planning workflow—with order submission permanently unavailable in the public preview.

## Professional description

I built this project to explore how a complex, data-rich workflow can remain understandable as its scope grows. The retained engine combines authenticated REST and WebSocket data, position and order reconciliation, fee-aware planning, state management, queue monitoring, and diagnostics.

For the public preview, I added a separate protection boundary that keeps the experience permanently in dry-run mode. Exact API-origin controls, external credential storage, hash-locked dependencies, an SBOM, a sealed file inventory, and automated checks give reviewers concrete ways to inspect the build and its boundaries.

The result is an advanced engineering study in API integration, information reconciliation, observability, and release governance. It complements the smaller 10×1¢ flagship by showing how the same discipline applies to a much larger system while keeping public interaction read-only.

## Case-study summary

**Opportunity:** Create a useful public learning experience from a large private prototype without exposing private data or enabling account changes.

**Approach:** Separate the public preview from production behavior, keep every interaction read-only, move credentials and state outside the repository, lock dependencies, seal the file inventory, and document the system's data flows and review points.

**Result:** A substantial technical showcase that highlights API integration, data reconciliation, operational diagnostics, privacy-aware packaging, and disciplined release preparation.
